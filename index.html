Configuration

You should require only one script tag on your html since Arenite can take care of loading your javascript. This is advisable as there is a mechanism within Arenite to run in different flavors with different files

(ex: debug with all source files and prod with minified versions)

There are two ways to configure you application using Arenite. You can use object configuration or annotation-based configuration. You can use both at the same time is such is desired.

Object

You can check the complete reference for the configuration object here

Dependencies

A minimal initial object based configuration is required to specify the resources.

    Arenite({
      context: {
        dependencies: {
          default: {
            sync: [
                ...
            ],
            async: [
              ...
            ]
          }
        }
      }
    });

dependencies is the configuration part where you define the script to load. Note there is a default entry in this element and that will be the files loaded by default. You can add any other sections you'd like and trigger those alternatives by appending the query param env to the url.

Example: ...index.html?env=debug will attempt to load scripts defined in a debug section.

Each dependency configuration contains two lists, sync and async which are used to list your scripts that need to be loaded synchronously or can be loaded asynchronously respectively.

Note that all sync scripts are loaded one after the other and only then the async scripst are loaded.

Imports

For any non-trivial web-application you can (and should) separate your configuration into several different files. this can then be specified in your main configuration (usually in index.html). 

    Arenite({
      imports: [
        {
          url: '.../sample.js',
          namespace: 'App.Sample'
        }
        ...
      ]
    });

namespace specifies the function to run to obtain the partial configuration. url is an optional setting that allows Arenite to fetch the source for App.Sample if it is not already defined (loaded from the dependencies for example). Note that only the content of context will be used.

Configuration Reference

Configuration object

Option Valid Values Default Description
expose true | false false Specifies if the Arenite instance should be exposed in the window object
imports array of import objects undefined The partial configurations to include
context context object undefined The context to load and wire

Import object

Option Valid Values Default Description
url url string undefined specifies the location of the script to load in case the function does not already exist
namespace function identifier string undefined defines a function that must return a Arenite configuration object

Context Object

Option Valid Values Default Description
dependencies dependencies object undefined Object that defines the dependencies to load per environment. Each key in this object represents an environment with that name and each value is a Dependency object
instances instances object undefined Object that defines the instances to create. Each key in this object represents an instance with that name and each value is an Instance object
start list of Execution object undefined List that contains the definition of what should be executed after the context has been wired

Dependency object

Option Valid Values Default Description
sync array of urls undefined List of dependency urls to load in sequence
async array of urls undefined List of dependency urls to load in parallel after sync

Instance object

Option Valid Values Default Description
extension true | false false defines if the instance should extend the arenite object with the return value
factory true | false false defines if a new instance should be created each time it is requested
namespace function identifier string undefined defines a function that will be invoked with the resolved args
args List of Argument object undefined determines the arguments to use when invoking the function specified in namespace
init Execution object undefined Defines a function that should be invoked as initialization after the creation of the instance.

Argument object

The argument object should only have one of it's options filled in.

Option Valid Values Default Description
value any javascript type undefined A value
ref string with name of referenced object undefined An instance registered with the provided name
func function undefined A function, as is
exec function undefined The result of the function at invocation time

Execution object

Option Valid Values Default Description
instance name of the registered instance undefined The instance to use. In the case of an init, this is ignored and the current instance is used.
func function or string with function name undefined If func is a function it will be called with args. If it's a string it needs to be accompanied by the instance name unless it's within an init.
args List of Argument object undefined determines the arguments to use when invoking the function specified in func
